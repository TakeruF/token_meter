# Contributing

Contributions are welcome through GitHub pull requests.

- Do not include prompts, responses, credentials, real user paths, or other
  personal data in code, tests, fixtures, issues, or logs.
- Keep fixtures anonymous and limited to the fields required to reproduce the
  parser or aggregation behavior.
- Run the tests relevant to the changed platform and document the result in the
  pull request.
- Changes from non-committers require maintainer review before merge.
- Release workflows, package manifests, installers, and dependency changes
  require explicit review because they affect signed artifacts.

The responsibilities used for code signing are documented in
[the Code signing policy](CODE_SIGNING.md).

Participation in this project is governed by the
[Code of Conduct](CODE_OF_CONDUCT.md).
