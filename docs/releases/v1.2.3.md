# Token Meter 1.2.3

## English

This update adds a one-click sign-in shortcut for expired Claude Code sessions, and brings both time windows to the small widget.

### Changes

- Added a "Sign in to Claude Code" button wherever Token Meter reports an expired or unauthorized Claude session (Setup and the menu bar popover). It opens Terminal and runs `claude` for you — you only need to type `/login` once it starts. The button only appears once Token Meter has confirmed `claude` is actually on your PATH, and Token Meter still never reads, stores, or refreshes your Claude credentials itself.
- The small (2×2) widget now shows both the 5-hour and weekly reset countdowns for Claude and Codex, stacked compactly, instead of just the 5-hour reset for Claude alone.

### System Requirements

- macOS 14.0 or later
- Apple Silicon / Intel Mac (Universal Binary)

### Installation

Unzip `TokenMeter-1.2.3.zip` and move `TokenMeter.app` to `/Applications`. If you are already using Token Meter, you can update from the menu by clicking "Check for Updates…".

The app is Developer ID signed and notarized by Apple. The update ZIP is also verified by Sparkle's EdDSA signature.

- 小サイズ（2×2）のウィジェットに、ClaudeとCodex両方の5時間枠・週間枠のリセットまでの時間を、コンパクトに縦2行で表示するようにしました。以前はClaudeの5時間枠のみの表示でした

### 動作環境

- macOS 14.0以降
- Apple Silicon / Intel Mac（Universal Binary）

- macOS 14.0 或更高版本

## 한국어

Claude Code 로그인이 만료되었을 때 원클릭으로 다시 로그인할 수 있는 버튼을 추가하고, 작은 위젯에 두 시간 창을 모두 표시하도록 개선한 업데이트입니다.

### 변경 내용

- Token Meter가 Claude 세션 만료 또는 인증 실패를 알리는 위치(설정 화면, 메뉴 막대 팝오버)에 'Claude Code에 로그인' 버튼을 추가했습니다. 누르면 터미널이 열리고 `claude`가 자동으로 실행되며, 시작되면 `/login`만 입력하면 됩니다. 이 버튼은 Token Meter가 PATH에서 `claude` 명령을 실제로 확인했을 때만 표시되며, Token Meter 자체는 여전히 Claude 인증 정보를 읽거나 저장하거나 갱신하지 않습니다.
- 작은(2×2) 위젯이 이제 Claude와 Codex 모두의 5시간 및 주간 재설정 카운트다운을 컴팩트하게 함께 표시합니다. 이전에는 Claude의 5시간 재설정만 표시했습니다.

### 작동 환경

- macOS 14.0 이상
- Apple Silicon / Intel Mac (Universal Binary)

### 설치 방법

`TokenMeter-1.2.3.zip` 압축을 풀고 `TokenMeter.app`을 `/Applications`로 이동하십시오. 이미 Token Meter를 사용 중인 경우 앱 내 'Check for Updates…'를 통해 업데이트할 수 있습니다.

앱은 Developer ID로 서명되었으며 Apple의 공증(Notarization)을 받았습니다. 업데이트용 ZIP은 Sparkle의 EdDSA 서명으로도 검증됩니다.
