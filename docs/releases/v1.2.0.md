# Token Meter 1.2.0

## English

This update adds GitHub Copilot CLI tracking, a setup step that verifies the connection actually works, and longer comparison periods on the dashboard.

### Changes

- Added GitHub Copilot CLI usage tracking. It counts tokens only, and its models overlap with Claude Code, so it is off by default — turn it on in Settings if you use Copilot in the terminal.
- Added a connection check to the setup guide. It verifies that usage can actually be read for what you picked, not just that a tool is installed, and tells you exactly what to fix — signed out, no session log yet, or macOS blocking access. Press "Re-check" after fixing, or finish setup now and connect later while Token Meter keeps looking in the background.
- Added a 90-day range to the dashboard, along with "vs yesterday", "vs prev 7 days", "vs prev 30 days", and "vs prev 90 days" comparisons.
- Token Meter now notices when Claude Code's sign-in has expired and tells you to run `/login` instead of silently showing no plan usage.

### System Requirements

- macOS 14.0 or later
- Apple Silicon / Intel Mac (Universal Binary)

### Installation

Unzip `TokenMeter-1.2.0.zip` and move `TokenMeter.app` to `/Applications`. If you are already using Token Meter, you can update from the menu by clicking "Check for Updates…".

The app is Developer ID signed and notarized by Apple. The update ZIP is also verified by Sparkle's EdDSA signature.

## 日本語

GitHub Copilot CLIの集計、接続を実際に確認する初期設定ステップ、ダッシュボードの比較期間の拡張を追加するアップデートです。

### 変更内容

- GitHub Copilot CLIの使用量集計に対応。トークン数のみを集計し、モデルはClaude Codeと重複するため、デフォルトはオフです。ターミナルでCopilotを使う場合は設定からオンにしてください
- 初期設定に接続確認を追加。ツールがインストールされているかどうかだけでなく、選んだ対象の使用量を実際に読み取れるかを確認し、未ログイン・セッションログ未作成・macOSによるアクセスのブロックなど、直すべき内容を具体的に表示。修正後に「再確認」を押せます。Token Meterはバックグラウンドでも探し続けるため、今は設定を終えて後で接続しても構いません
- ダッシュボードに90日間の範囲と、「前日比」「前7日比」「前30日比」「前90日比」の比較を追加
- Claude Codeのサインイン失効を検知し、プラン使用量が表示されないまま放置せず、`/login`での再ログインを案内するように

### 動作環境

- macOS 14.0以降
- Apple Silicon / Intel Mac（Universal Binary）

### インストール

`TokenMeter-1.2.0.zip`を展開し、`TokenMeter.app`を`/Applications`へ移動してください。既にToken Meterを使用している場合は、アプリ内の「Check for Updates…」から更新できます。

アプリはDeveloper IDで署名し、AppleのNotary Serviceで公証しています。更新ZIPはSparkleのEdDSA署名でも検証されます。

## 中文

此更新新增了 GitHub Copilot CLI 的统计、可实际验证连接的初始设置步骤，以及仪表盘上更长的对比周期。

### 变更内容

- 新增 GitHub Copilot CLI 使用量统计。仅统计 Token 数量，且其模型与 Claude Code 重复，因此默认关闭。如果您在终端中使用 Copilot，请在设置中开启。
- 在初始设置中新增连接确认。不仅检查工具是否已安装，还会确认能否实际读取所选对象的使用量，并明确提示需要修复的内容——未登录、尚无会话日志或 macOS 阻止访问。修复后可点击“重新确认”。Token Meter 也会在后台持续查找，因此您可以先完成设置，稍后再连接。
- 在仪表盘中新增 90 天范围，以及“与昨日对比”“与前 7 天对比”“与前 30 天对比”“与前 90 天对比”。
- 现在会检测 Claude Code 登录失效的情况，并提示运行 `/login` 重新登录，而不是始终不显示套餐使用量。

### 运行环境

- macOS 14.0 或更高版本
- Apple Silicon / Intel Mac（通用二进制）

### 安装方法

解压 `TokenMeter-1.2.0.zip`，然后将 `TokenMeter.app` 移至 `/Applications`。如果您已在使用 Token Meter，可通过应用内的“Check for Updates…”进行更新。

应用已使用 Developer ID 签名并已通过 Apple 公证。更新 ZIP 包也通过 Sparkle 的 EdDSA 签名进行验证。

## 한국어

GitHub Copilot CLI 집계, 연결을 실제로 확인하는 초기 설정 단계, 대시보드의 비교 기간 확장을 추가하는 업데이트입니다.

### 변경 내용

- GitHub Copilot CLI 사용량 집계를 지원. 토큰 수만 집계하며 모델이 Claude Code와 중복되므로 기본값은 꺼짐입니다. 터미널에서 Copilot을 사용하는 경우 설정에서 켜십시오.
- 초기 설정에 연결 확인을 추가. 도구 설치 여부뿐만 아니라 선택한 대상의 사용량을 실제로 읽을 수 있는지 확인하고, 미로그인·세션 로그 없음·macOS의 접근 차단 등 수정해야 할 내용을 구체적으로 안내합니다. 수정 후 '다시 확인'을 누를 수 있습니다. Token Meter는 백그라운드에서도 계속 확인하므로 지금 설정을 마치고 나중에 연결해도 됩니다.
- 대시보드에 90일 범위와 '전일 대비', '이전 7일 대비', '이전 30일 대비', '이전 90일 대비' 비교를 추가.
- Claude Code의 로그인 만료를 감지하여 플랜 사용량이 표시되지 않은 채로 두지 않고 `/login` 재로그인을 안내하도록 개선.

### 작동 환경

- macOS 14.0 이상
- Apple Silicon / Intel Mac (Universal Binary)

### 설치 방법

`TokenMeter-1.2.0.zip` 압축을 풀고 `TokenMeter.app`을 `/Applications`로 이동하십시오. 이미 Token Meter를 사용 중인 경우 앱 내 'Check for Updates…'를 통해 업데이트할 수 있습니다.

앱은 Developer ID로 서명되었으며 Apple의 공증(Notarization)을 받았습니다. 업데이트용 ZIP은 Sparkle의 EdDSA 서명으로도 검증됩니다.
