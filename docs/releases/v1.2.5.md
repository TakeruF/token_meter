# Token Meter 1.2.5

## English

This update turns "your Claude Code sign-in expired" into a one-click fix, and stops the widget from getting stuck on its empty state.

### Changes

- Added a one-click way out of every Claude connection problem. Where Token Meter reports an expired sign-in, a missing Claude Code install, or a denied Keychain item — the menu bar card, Setup, and the setup guide — it now offers a button that does the next step for you: opening Terminal with `claude` ready for `/login`, opening the download page, or performing the single Keychain read that makes macOS show its approval dialog.
- Every one of those steps asks first. A popup states exactly what is about to happen before anything opens, and each popup also links to the written manual steps for anyone who would rather do it by hand. When a step turns out to reveal a different problem — the Keychain read succeeds but the token has expired, or `claude` is not installed after all — Token Meter offers the next step instead of dead-ending.
- The Keychain approval can now be granted deliberately. Right after you enable Claude Pro / Max usage, in the setup guide and in Settings, a button performs the credential read on demand, so macOS asks for approval one click after you asked for it rather than out of a background refresh. Choosing "Always Allow" in the system dialog stops the prompts for good.
- Added a documentation page covering all of it by hand: checking the install, `/login`, the Keychain approval, and what to do if you denied it. It is linked from the app and from the setup section of the website.
- Fixed the widget showing "No data" after being added or resized. macOS renders the placeholder while a widget waits for its first timeline, and the placeholder held no data; it now shows the last good snapshot. Widget reloads are also rate-limited, so a burst of refreshes at launch no longer spends WidgetKit's reload budget and leaves a later reload dropped.
- Changed the Clear widget background to a translucent frosted material. It now genuinely shows the wallpaper through the widget in every appearance mode, and works on every supported macOS version — previously it required macOS 26 and looked opaque unless the system's widget-dimming setting was on.

### System Requirements

- macOS 14.0 or later
- Apple Silicon / Intel Mac (Universal Binary)

### Installation

Unzip `TokenMeter-1.2.5.zip` and move `TokenMeter.app` to `/Applications`. If you are already using Token Meter, you can update from the menu by clicking "Check for Updates…".

The app is Developer ID signed and notarized by Apple. The update ZIP is also verified by Sparkle's EdDSA signature.

## 日本語

「Claude Codeのサインインが失効しました」をワンクリックで解決できるようにし、ウィジェットが空表示のまま固まる問題を修正したアップデートです。

### 変更内容

- Claudeの接続トラブルを、ワンクリックで解消できるようにしました。サインインの失効、Claude Codeの未インストール、Keychainのアクセス拒否をToken Meterが表示する場所（メニューバーのカード、セットアップ、初期設定ガイド）に、次の一手を代行するボタンを追加しました。`claude`を実行したターミナルを開いて`/login`できる状態にする、ダウンロードページを開く、macOSの許可ダイアログを出すためのKeychain読み取りを1回だけ実行する、のいずれかを行います。
- どの操作も必ず先に確認します。何かが開く前に「これから何が起きるか」を説明するポップアップを表示し、そのポップアップからは手動手順のドキュメントも開けます。実行して別の問題が判明した場合（Keychainは読めたがトークンが失効していた、`claude`が実際には入っていなかったなど）は、行き止まりにせず次の手順を案内します。
- Keychainの許可を、自分の意思で出せるようにしました。Claude Pro / Max使用量を有効にした直後、初期設定ガイド、設定画面にボタンを用意し、その場で資格情報の読み取りを実行します。バックグラウンドの更新中に突然macOSの許可ダイアログが出るのではなく、クリックした直後に出るようになります。システムのダイアログで「常に許可」を選べば、以後は聞かれません。
- 上記をすべて手動で行うためのドキュメントを追加しました。インストールの確認、`/login`、Keychainの許可、拒否してしまった場合の対処までを記載し、アプリ内とWebサイトのセットアップ節からリンクしています。
- ウィジェットを追加・リサイズした後に「データなし」と表示される問題を修正しました。最初のタイムラインが届くまでmacOSはプレースホルダーを描画しますが、そこにデータがありませんでした。今後は直前の有効なスナップショットを表示します。あわせてウィジェットのリロードを間引くようにしたので、起動直後の連続更新でWidgetKitのリロード上限を使い切り、後のリロードが破棄されることがなくなります。
- ウィジェットのクリア背景を、半透明のすりガラス素材に変更しました。どの表示モードでも壁紙が実際に透けて見えるようになり、サポート対象のすべてのmacOSで利用できます（従来はmacOS 26以降が必要で、システムのウィジェット減光設定がオンのときしか透けて見えませんでした）。

### 動作環境

- macOS 14.0以降
- Apple Silicon / Intel Mac（Universal Binary）

### インストール

`TokenMeter-1.2.5.zip`を展開し、`TokenMeter.app`を`/Applications`へ移動してください。既にToken Meterを使用している場合は、アプリ内の「Check for Updates…」から更新できます。

アプリはDeveloper IDで署名し、AppleのNotary Serviceで公証しています。更新ZIPはSparkleのEdDSA署名でも検証されます。

## 中文

此更新让“Claude Code 登录已过期”可以一键解决，并修复了小组件卡在空白状态的问题。

### 变更内容

- 为每一种 Claude 连接问题提供了一键解决方式。在 Token Meter 提示登录过期、未安装 Claude Code 或钥匙串被拒绝的位置（菜单栏卡片、设置页、初始设置向导），现在都会显示一个按钮来代你完成下一步：打开已运行 `claude` 的终端以便输入 `/login`、打开下载页面，或执行一次钥匙串读取以触发 macOS 的授权对话框。
- 每一步都会先征求确认。在任何东西被打开之前，弹窗会明确说明即将发生什么，并且弹窗中也可以打开手动步骤文档。如果执行后发现的是另一个问题（钥匙串读取成功但令牌已过期，或实际上并未安装 `claude`），Token Meter 会继续给出下一步，而不是走进死胡同。
- 钥匙串授权现在可以主动完成。在启用 Claude Pro / Max 用量之后、初始设置向导中以及设置页中，都提供了按钮来按需执行凭据读取，因此 macOS 的授权对话框会在你点击之后立即出现，而不是从后台刷新中突然弹出。在系统对话框中选择“始终允许”即可不再被询问。
- 新增了一份手动操作文档：确认安装、`/login`、钥匙串授权，以及误点拒绝后的处理方法。应用内和网站的设置章节均已链接该文档。
- 修复了添加或调整小组件大小后显示“无数据”的问题。在小组件等待首个时间线期间，macOS 会渲染占位视图，而该视图此前没有数据；现在会显示上一次的有效快照。同时对小组件刷新做了限流，启动时的密集刷新不会再耗尽 WidgetKit 的刷新配额而导致后续刷新被丢弃。
- 将小组件的透明背景改为半透明磨砂材质。现在在所有外观模式下都能真正透出壁纸，并且在所有受支持的 macOS 版本上都可使用（此前需要 macOS 26，且只有在系统的小组件变暗设置开启时才会呈现透明效果）。

### 运行环境

- macOS 14.0 或更高版本
- Apple Silicon / Intel Mac（通用二进制）

### 安装方法

解压 `TokenMeter-1.2.5.zip`，然后将 `TokenMeter.app` 移至 `/Applications`。如果您已在使用 Token Meter，可通过应用内的“Check for Updates…”进行更新。

应用已使用 Developer ID 签名并已通过 Apple 公证。更新 ZIP 包也通过 Sparkle 的 EdDSA 签名进行验证。

## 한국어

“Claude Code 로그인이 만료되었습니다”를 한 번의 클릭으로 해결할 수 있게 하고, 위젯이 빈 상태로 멈추는 문제를 수정한 업데이트입니다.

### 변경 내용

- 모든 Claude 연결 문제에 한 번의 클릭으로 대응할 수 있게 했습니다. Token Meter가 로그인 만료, Claude Code 미설치, 키체인 접근 거부를 알려 주는 위치(메뉴 막대 카드, 설정 페이지, 초기 설정 가이드)에 다음 단계를 대신 수행하는 버튼을 추가했습니다. `claude`가 실행된 터미널을 열어 `/login`을 입력할 수 있게 하거나, 다운로드 페이지를 열거나, macOS 승인 대화상자를 띄우는 키체인 읽기를 한 번 실행합니다.
- 모든 단계는 먼저 확인을 받습니다. 무언가가 열리기 전에 무슨 일이 일어날지 정확히 알려 주는 팝업이 표시되며, 그 팝업에서 수동 단계 문서도 열 수 있습니다. 실행 결과 다른 문제가 드러나면(키체인은 읽었지만 토큰이 만료된 경우, `claude`가 실제로는 설치되어 있지 않은 경우) 막다른 길 대신 다음 단계를 제안합니다.
- 키체인 승인을 직접 요청할 수 있게 했습니다. Claude Pro / Max 사용량을 켠 직후, 초기 설정 가이드, 설정 화면에 버튼을 두어 그 자리에서 자격 증명을 읽습니다. 백그라운드 새로 고침 중에 갑자기 뜨는 대신, 클릭한 직후에 macOS가 승인을 묻습니다. 시스템 대화상자에서 “항상 허용”을 선택하면 다시 묻지 않습니다.
- 이 모든 과정을 직접 수행하기 위한 문서를 추가했습니다. 설치 확인, `/login`, 키체인 승인, 거부했을 때의 복구 방법을 담았으며 앱과 웹사이트 설정 섹션에서 링크됩니다.
- 위젯을 추가하거나 크기를 바꾼 뒤 “데이터 없음”이 표시되던 문제를 수정했습니다. 첫 타임라인을 기다리는 동안 macOS는 플레이스홀더를 그리는데 여기에 데이터가 없었습니다. 이제 마지막으로 유효했던 스냅샷을 표시합니다. 또한 위젯 새로 고침을 제한해, 실행 직후의 연속 갱신이 WidgetKit의 새로 고침 예산을 소진해 이후 갱신이 무시되는 일이 없도록 했습니다.
- 위젯의 클리어 배경을 반투명 서리 재질로 변경했습니다. 이제 모든 표시 모드에서 배경화면이 실제로 비쳐 보이며, 지원되는 모든 macOS 버전에서 사용할 수 있습니다(이전에는 macOS 26 이상이 필요했고, 시스템의 위젯 흐리게 설정이 켜져 있을 때만 투명하게 보였습니다).
