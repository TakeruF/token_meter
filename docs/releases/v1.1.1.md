# Token Meter 1.1.1

## English

This patch update fixes the rendering size of brand logos in the menu bar's Compact mode.

### Changes

- Renders Claude and Codex logos as AppKit images specific to Status Items.
- Fixed logic to set logical sizes in advance, preventing intrinsic sizes of 512px source images from propagating to the menu bar.
- Treated Claude and Codex logos as macOS template images to adapt to light/dark modes and selection states.
- Unified previews in Settings, Setup, and onboarding with actual menu bar rendering.

### System Requirements

- macOS 14.0 or later
- Apple Silicon / Intel Mac (Universal Binary)

### Installation

Unzip `TokenMeter-1.1.1.zip` and move `TokenMeter.app` to `/Applications`. If you are already using Token Meter, you can update from the menu by clicking "Check for Updates…".

The app is Developer ID signed and notarized by Apple. The update ZIP is also verified by Sparkle's EdDSA signature.

## 中文

此补丁更新修复了菜单栏紧凑 (Compact) 模式下品牌图标的渲染大小。

### 变更内容

- 将 Claude 和 Codex 图标绘制为适用于状态栏 (Status Item) 的 AppKit 图像。
- 提前固定逻辑大小，防止 512px 源图片的原始大小 (intrinsic size) 传播到菜单栏。
- 将 Claude 和 Codex 图标视为 macOS 的模版 (template) 图像，使其自适应亮色/暗色模式以及选中状态。
- 将设置 (Settings)、初始引导和设置向导中的预览与实际菜单栏显示保持一致。

### 运行环境

- macOS 14.0 或更高版本
- Apple Silicon / Intel Mac（通用二进制）

### 安装方法

解压 `TokenMeter-1.1.1.zip`，然后将 `TokenMeter.app` 移至 `/Applications`。如果您已在使用 Token Meter，可通过应用内的“Check for Updates…”进行更新。

应用已使用 Developer ID 签名并已通过 Apple 公证。更新 ZIP 包也通过 Sparkle 的 EdDSA 签名进行验证。

## 한국어

메뉴 막대의 컴팩트(Compact) 표시 모드에서 브랜드 로고의 렌더링 크기를 수정하는 패치 업데이트입니다.

### 변경 내용

- Claude 및 Codex 로고를 상태 항목(Status Item) 전용 AppKit 이미지로 그리도록 수정.
- 512px 원본 이미지의 원래 크기(intrinsic size)가 메뉴 막대로 전파되지 않도록 논리적 크기를 사전에 고정.
- Claude 및 Codex 로고를 macOS 템플릿(template) 이미지로 취급하여 라이트/다크 모드 및 선택 상태에 연동.
- 설정(Settings), Setup, 온보딩 가이드의 미리보기를 실제 메뉴 막대 렌더링 결과와 통일.

### 작동 환경

- macOS 14.0 이상
- Apple Silicon / Intel Mac (Universal Binary)

### 설치 방법

`TokenMeter-1.1.1.zip` 압축을 풀고 `TokenMeter.app`을 `/Applications`로 이동하십시오. 이미 Token Meter를 사용 중인 경우 앱 내 'Check for Updates…'를 통해 업데이트할 수 있습니다.

앱은 Developer ID로 서명되었으며 Apple의 공증(Notarization)을 받았습니다. 업데이트용 ZIP은 Sparkle의 EdDSA 서명으로도 검증됩니다.
