# Token Meter 1.2.2

## English

This update fixes a reset notification that named the wrong limit, and makes every "Today" in the app agree. Codex and Copilot token counts were also being overstated.

### Changes

- Fixed reset notifications naming the wrong limit. When the 5-hour limit rolled over, the weekly limit often became the tightest one left, and the notice read "Weekly quota reset" — while the weekly limit had not moved at all. Worse, that mislabelled notice then suppressed the genuine 5-hour notices for the rest of the week. Each limit is now tracked on its own.
- Fixed Codex and Copilot token counts being overstated. Codex reports reasoning tokens inside its output count and cached tokens inside its input count, but the app added them on top, so "work" could come out higher than the total it is part of — Codex showed 26.23M of 26.20M. Claude Code's figures were correct and are unchanged.
- Made "Today" mean the same thing everywhere. The dashboard showed real work while the menu bar, the widget and the widget's chart showed cache-inclusive totals — the same label with numbers up to 50x apart. Every surface now leads with real work and shows the total beside it.
- The 5-hour, weekly and "Last 7 days" rows now show real work with their total, matching the cards next to them. Previously "Last 7 days" read 157M directly above "Today" at 1.19M, which invited an impossible comparison.

### System Requirements

- macOS 14.0 or later
- Apple Silicon / Intel Mac (Universal Binary)

### Installation

Unzip `TokenMeter-1.2.2.zip` and move `TokenMeter.app` to `/Applications`. If you are already using Token Meter, you can update from the menu by clicking "Check for Updates…".

The app is Developer ID signed and notarized by Apple. The update ZIP is also verified by Sparkle's EdDSA signature.

### 変更内容

- リセット通知が誤った上限を報告していた問題を修正。5時間の上限がリセットされると、残っている上限のうち週次が最も少なくなることが多く、週次はまったく動いていないのに「週次の上限がリセットされました」と通知していました。さらに、その誤った通知が以降1週間、本物の5時間リセット通知を抑制していました。各上限を個別に追跡するように変更しています
- CodexとCopilotのトークン数が過大に表示される問題を修正。Codexは推論トークンを出力に、キャッシュを入力に含めて報告しますが、アプリはそれらを別途加算していたため、「実作業」が総量を上回ることがありました（Codexで26.23M / 総量26.20M）。Claude Codeの数値は正しく、変更ありません
- 「今日」の意味をすべての画面で統一。ダッシュボードは実作業量を、メニューバー・ウィジェット・ウィジェットのグラフはキャッシュ込みの総量を表示しており、同じラベルで最大50倍異なる数字が出ていました。すべての画面で実作業量を主として表示し、総量を併記します
- 5時間枠・週次枠・「直近7日」の行も、実作業量と総量を併記するように変更。隣のカードと表記が揃います。以前は「直近7日 157M」のすぐ下に「今日 1.19M」が並び、比較できない数字が並んでいました

## 中文

此更新修复了重置通知报告错误上限的问题，并统一了应用内“今天”的数值。同时修复了 Codex 与 Copilot 的 Token 数量被高估的问题。

### 变更内容

- 修复重置通知报告错误上限的问题。当 5 小时上限重置后，周上限往往成为剩余额度最少的一项，于是通知显示“周配额已重置”——而周上限其实完全没有变动。更严重的是，该错误通知随后会在一周内抑制真正的 5 小时重置通知。现已改为分别追踪每个上限。
- 修复 Codex 与 Copilot 的 Token 数量被高估的问题。Codex 将推理 Token 计入输出、将缓存 Token 计入输入进行报告，但应用却额外累加了它们，导致“实际工作量”可能超过其所属的总量（Codex 显示 26.23M / 总量 26.20M）。Claude Code 的数值本就正确，未作更改。
- 统一“今天”在各处的含义。仪表盘显示实际工作量，而菜单栏、小组件及小组件图表显示含缓存的总量——相同的标签，数字却相差最多 50 倍。现在所有界面均以实际工作量为主，并在旁边显示总量。
- 5 小时、周以及“最近 7 天”各行现在也会显示实际工作量与总量，与相邻卡片的表示保持一致。此前“最近 7 天 157M”就显示在“今天 1.19M”的正上方，导致无法进行有意义的比较。

### 运行环境

- macOS 14.0 或更高版本
- Apple Silicon / Intel Mac（通用二进制）

### 安装方法

解压 `TokenMeter-1.2.2.zip`，然后将 `TokenMeter.app` 移至 `/Applications`。如果您已在使用 Token Meter，可通过应用内的“Check for Updates…”进行更新。

应用已使用 Developer ID 签名并已通过 Apple 公证。更新 ZIP 包也通过 Sparkle 的 EdDSA 签名进行验证。

## 한국어

재설정 알림이 잘못된 한도를 알리던 문제를 수정하고, 앱 내 '오늘' 수치를 통일하는 업데이트입니다. Codex와 Copilot의 토큰 수가 과다하게 표시되던 문제도 함께 수정했습니다.

### 변경 내용

- 재설정 알림이 잘못된 한도를 알리던 문제를 수정. 5시간 한도가 재설정되면 남은 한도 중 주간 한도가 가장 적어지는 경우가 많아, 주간 한도는 전혀 움직이지 않았는데도 '주간 할당량이 재설정되었습니다'라고 알렸습니다. 더욱이 그 잘못된 알림이 이후 일주일 동안 실제 5시간 재설정 알림을 억제했습니다. 이제 각 한도를 개별적으로 추적합니다.
- Codex와 Copilot의 토큰 수가 과다하게 표시되던 문제를 수정. Codex는 추론 토큰을 출력에, 캐시 토큰을 입력에 포함해 보고하지만 앱이 이를 별도로 더해, '실제 작업량'이 그것이 속한 총량을 넘어서는 경우가 있었습니다(Codex에서 26.23M / 총량 26.20M). Claude Code의 수치는 정확했으며 변경되지 않았습니다.
- '오늘'의 의미를 모든 화면에서 통일. 대시보드는 실제 작업량을, 메뉴 막대·위젯·위젯 차트는 캐시를 포함한 총량을 표시해 같은 레이블에 최대 50배 차이 나는 숫자가 나왔습니다. 이제 모든 화면이 실제 작업량을 앞세우고 그 옆에 총량을 함께 보여줍니다.
- 5시간, 주간, '최근 7일' 행도 실제 작업량과 총량을 함께 표시하여 옆의 카드와 표기가 일치합니다. 이전에는 '최근 7일 157M' 바로 아래에 '오늘 1.19M'이 놓여, 비교할 수 없는 숫자가 나란히 있었습니다.

### 작동 환경

- macOS 14.0 이상
- Apple Silicon / Intel Mac (Universal Binary)

### 설치 방법

`TokenMeter-1.2.2.zip` 압축을 풀고 `TokenMeter.app`을 `/Applications`로 이동하십시오. 이미 Token Meter를 사용 중인 경우 앱 내 'Check for Updates…'를 통해 업데이트할 수 있습니다.

앱은 Developer ID로 서명되었으며 Apple의 공증(Notarization)을 받았습니다. 업데이트용 ZIP은 Sparkle의 EdDSA 서명으로도 검증됩니다.
